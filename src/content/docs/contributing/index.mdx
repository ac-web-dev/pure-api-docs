Contributions to this documentation are welcome.
This section covers everything needed to begin.

## Where to start

For small corrections, use the `Edit page` button at the bottom of any page.
GitHub will help you create a fork of the repository, from which u can pull request a change.

For larger contributions, including guides, references, structural changes etc.
read the following guides before opening a pull request:

- [Setup](/contributing/setup) and get the project running locally.
- [Adding pages](/contributing/adding-pages) for file naming, structuring and reference templates.
- [Style guide](/contributing/style-guide) for prose and formatting expectations.
- [Versioning](/contributing/versioning) to learn how to manage content across versions of Pure.

:::note
Pull requests should not be too large.
In the case that a pull request does not meet guidelines, the contributor may be asked to make changes or split it into smaller PRs.
:::

## Reporting issues

Follow [this](https://github.com/ac-web-dev/pure-api-docs/issues) link to open an issue on GitHub.
